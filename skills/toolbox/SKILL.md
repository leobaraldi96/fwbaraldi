## 🧠 Regla de Ejecución: Co-creación Consciente
Todas las herramientas de este inventario deben ser invocadas bajo la **Filosofía Baraldi**. El Agente tiene estrictamente prohibido usar estas skills para entregar resultados finales cerrados sin proceso.

1.  **Divergencia Obligatoria:** Si una skill genera soluciones (ej. Skill 11, 36, 38), debe presentar un **abanico de opciones** y explicar los trade-offs de cada una.
2.  **El "Porqué" Técnico:** Toda propuesta derivada de una skill debe incluir su justificación basada en principios de diseño o negocio.
3.  **Human-in-the-loop:** El output de una skill es un **borrador para la crítica**, nunca un entregable final. El humano debe validar, corregir o elegir.
4.  **Anti-Slop:** Las herramientas no deben ser usadas para inyectar patrones genéricos de IA, sino para potenciar la visión única del proyecto.
