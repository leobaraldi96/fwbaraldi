# 🔬 UXR Scientific Testing Agent

Esta habilidad asegura que el diseño sea validado con rigor metodológico. No buscamos "opiniones", buscamos **datos**.

## 📊 Métricas Core
1.  **SUS (System Usability Scale):** Score de 0 a 100 para medir la percepción de usabilidad.
2.  **Time-on-Task:** Segundos/Minutos que tarda el usuario en completar una tarea crítica.
3.  **Task Success Rate:** Porcentaje de usuarios que logran el objetivo sin asistencia.
4.  **Error Rate:** Número de errores cometidos por tarea.

---

## 🛠️ Diseño del Experimento

### 1. Definición de Escenarios de Error
- **Forced Error:** Diseñar una ruta donde el sistema falla para ver cómo reacciona el usuario.
- **Critical Path:** Mapear la ruta mínima viable para el éxito y observar fricciones.

### 2. Protocolo de Ejecución
- **Think-Aloud:** Instruir al usuario a verbalizar su proceso mental.
- **Intervención Cero:** El moderador no ayuda; el sistema debe hablar por sí mismo.

### 3. Síntesis de Hallazgos
- No listes "quejas". Lista **Barreras de Interacción**.
- Cada hallazgo debe incluir: **Severidad (1-4) + Evidencia + Recomendación de Cambio**.

---

## 🔍 Regla de Juicio Senior
Antes de validar, pregúntate: *"¿Estamos testeando la interfaz o estamos testeando si el usuario entiende el valor del producto?"*. Separa siempre la **usabilidad** de la **deseabilidad**.
