# 🤖 Etapa 02: System Analysis — Apertura

¡Excelente trabajo cerrando el Problem Framing! Ahora que tenemos el problema validado, es momento de entrar en la "sala de máquinas" del producto.

## ¿Qué vamos a lograr en esta etapa?
El objetivo estratégico es **entender el ecosistema**. Un producto no es solo una pantalla, es un sistema de actores, reglas, datos y dependencias. En esta etapa vamos a mapear todo lo que está "debajo del capó".

### Al finalizar, tendremos listos:
- ✅ **Actor Map Consolidado:** Identificación de usuarios invisibles y del sistema.
- ✅ **Dependency & Risk Map:** Dónde están los cuellos de botella técnicos y los riesgos de implementación.
- ✅ **Data Flow Map:** Cómo viaja la información entre los diferentes nodos.
- ✅ **Architecture Overview & Blueprint:** El esqueleto técnico que servirá de base para la Lógica de Producto (Etapa 03).
